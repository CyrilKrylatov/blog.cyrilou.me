Sublime Text 3 : réinstallation des plugins

03 July 2013

Vous le savez peut-être, et si vous ne le saviez pas je vais vous le dire, ça vous évitera de l’ignorer : Sublime Text s’est vu sortir sa version 3 (version public beta).

TL;DR: $ cd ~/Library/Application\ Support/Sublime\ Text\ 3/Packages/ && git clone https://github.com/wbond/sublime_package_control.git « Package Control » && cd « Package Control » && git checkout python 3

Et si comme moi vous avez fait la mise à jour à l’arrache (je prends le .app et je le remplace), vous verrez que vous n’aurez plus de plugins. Et oui, tout simplement. La raison à ça, c’est que le dossier où sont contenus les plugins n’est pas le même entre la version 2 et la 3. Enfin c’est ce que j’ai constaté. Si ça ne vous ai pas arrivé, tant mieux.

De ce fait, pour réinstaller les plugins, il vous faudra mettre la main dans le cambouis. Avant, pour installer ça, il fallait simplement glisser le .package dans le dossier « Installed Packages » et voilà MAIS PLUS MAINTENANT !

En effet, Sublime Text 3, pour le moment (espérons), n’extrait plus par défaut un .package téléchargé, il faudra donc passer par ch’git pour faire ça (assuming the fact that vous connaissez Git, cela va s’en dire). Voici la méthode pour réinstaller Sublime Package Control sur Sublime Text 3.

Il vous faudra ouvrir dans le terminal le dossier Preferences -> Browse Packages et taper les commandes suivantes :

1
2
3

git clone https://github.com/wbond/sublime_package_control.git "Package Control"
cd "Package Control"
git checkout python3

La première commande va cloner le projet dans un dossier « Package Control », la 2ème vous mettre dans ce dossier et la 3ème changer la branch du repo git fraichement clonée pour avoir celle « compatible » Sublime Text 3.

Il semble que pour le moment, il n’y ai que cette méthode pour installer des plugins, il ne faudra pas hésiter à aller sur les différentes sites / repo github (exemple avec SideBar Enhancements) pour voir si le plugin est directement compatible et si non, voir s’il y a une branche de créée avec tout ce qu’il faut dedans.
